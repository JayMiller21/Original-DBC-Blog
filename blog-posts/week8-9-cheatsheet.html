String

A string is a chain of characters that usually forms human readable words surrounded by either single or double quotes. A string can be empty, which would just be a pair of double/single quotes and no character inside. String manipulation is an important part of computer science, and there are a number of methods in Ruby that can be used on String class. These include but are not limited to the following.

String#print

Outputs the string to the console

puts "Howdy partner!" => "Howdy partner!"

String#puts

Outputs the string to the console followed by a line break.

puts "Howdy partner!" => "Howdy partner!"
#new line

String#concat

string1.concat(string2) appends string2 to the end of string1

"good ".concat("times") => "good times"

Alternatively, the '+' or '<<' also will perform a similar function and may even give you a slight performance boost!

"good " + "times" => "good times"
"good " << "times" => "good times"

String#ord

When performed on a single character, returns that character's ASCII value. When performed on a multi-character string, returns the ASCII value of the first letter in the string.

"a".ord => 97
"apple".ord => 97

String#to_sym

Returns a symbol object that cooresponds to the original string.

"coolbeans".to_sym => :coolbeans

For a string with spaces, the symbol name will be in quotes.

"cool beans".to_sym => :"cool beans"

Symbols are very similar to strings, execept that they cannot be changed once they are created.

foo = "fighters".to_sym
foo + " rool!" => Throws an Undefined Method Error

Integer

Integers are values that are represented rounded down to the nearest whole number. The following are some of Integer's most common methods.

Integer#to_s

Turns an integer into a string containing the digits as a corresponding series of characters.

963.to_s => "963"

Integer#chr

Returns the cooresponding character for the ASCII value.

97.chr => "a"

Mathematical Operations (+, -, *, /)

Preform the specified mathemaitical function. Return integers for operations involving only integers. Return floats for operations involving one or more floats.

1 + 2 => 3
1.0 + 2 => 3.0
5 - 2 => 3
5.0 - 2 => 3.0
1 * 2 => 3
1.0 * 2 => 3.0
5/2 => 2
5.0/2 => 2.5

Modulus (%)

The modulus operator returns the remainder of dividing one integer by another.

5 % 2 => 1

Array

An array is an ordered collection of elements that can be of any type. These elements are assigned a numeric position in the array that relates to their order. The first item in every array starts off with an index of 0 and each subsequent item's index is increased by 1.

Array#slice(index)

Or more commonly seen as Array[index], this will retrieve the element of an array that is in that position.

sweet_stuff = ["tree","flower","fauna","river","bridge"]
sweet_stuff[2] => "fauna"
sweet_stuff.slice(2) => "fauna"

Array#slice(start,length)

You can also give it a starting index, along with a number of subsequent elements to retrieve.

sweet_stuff[1,3] => ["flower","fauna","river"]

Array#slice(range)

Lastly, you can pass in a range of indicies and it will return that.

sweet_stuff[1..3] => ["flower","fauna","river"]

Array#push(item)

Push will insert an item as the last element of a given array.

test = [1,2,3,4]
test.push(5) => [1,2,3,4,5]

Array#pop

The opposite of push, pop will remove the last item from a given array and return it. It is also important to know that pop modifies the existing object.

test = [1,2,3,4]
test.pop => 4
test = [1,2,3]

Hash

A hash is collection of unordered key value pairs. In a nutshell, instead of using a regular array, a hash function maps various keys, which for the most part are human-readable alphanumeric strings, and connect them to indexs of an array. As it turns out, it is much quicker to see if a given key exists, versus having to check each item of the array to see if the desired item exists in there. As a result, hashes tend to be way way quicker for searching purposes than regualr arrays.

Accessing Values

You can get access to the values in a has similar to how you would from an array, execept you refer to the corresponding key as opposed to the index.

test_hash = {"name" => "Bob", "hobby" => "basketball"}
test_hash["name"] => "Bob"

How to access values with symbols for keys

With this synatx, keys are implicitly turned into symbols.

sym_hash = {name: "Bob", hobby: "basketball"}
sym_hash[:name] => "Bob"

Hash#each {|key,value| block}

Will iterate through each key,value pair of the hash and perform whatever code block is passed to it.

test_hash = {"name" => "Mike", "age" => "23", "city" => "Brooklyn"}
test_hash.each {|key, value| puts key + " " + value} => "name Mike", "\n","age 23", "\n", "city Brooklyn
